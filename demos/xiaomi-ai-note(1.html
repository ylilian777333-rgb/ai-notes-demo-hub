01 · 问题

记录工具的时代已经过去了

过去十年，笔记产品解决的是"怎么记"的问题。但今天用户真正面临的问题是三个：

记录成本太高——打开 App、新建、输入，流程依然繁琐。
记下来没有价值——大量笔记沉底，从不被再次调用。
数据极度分散——短信、截图、收藏、录音各在一处，无法整合。

AI 时代，记录的本质已经变了：
从「存储信息」→ 变成「积累个人数字资产」。
02 · 机会

小米天然具备做这件事的条件

普通笔记产品只有一个入口。小米有整个生态，而且两种生态能力是叠加的：

📱 手机生态
主动记录 + 上下文感知 + 被动数据打通
记录有入口，数据有来源
⌚ 非手机生态
手表、手环、耳机、车机、眼镜（未来）
无感化记录——用户甚至不需要主动触发

这是其他笔记产品无法复制的能力。把所有设备的数据统一接入一个系统，就形成了别人做不到的东西：

Personal Memory Layer
个人数字记忆层——小米生态独有的能力。

05 · 生态位

AI Note 在小米生态中的位置

AI Note 的价值不在于自身功能有多强，而在于它在生态中占据的形成剂位置——它把所有设备、所有入口产生的数据，统一沉淀到同一个地方。

📥 数据汇聚
小爱同学触发、长按 AI 键、手表手环无感记录……所有入口的内容，都流入 AI Note
📤 对外开放
通过 MCP 方式对外暴露接口，小爱、系统搜索、AI 推荐、第三方 Agent 均可调用
AI Note = 小米生态的 Personal Memory API
一个可感知的个人数字资产库，服务于整个系统。
06 · 战略价值

这件事对小米意味着什么

01
降低记录门槛，积累数据资产
记录变得无感、自然，用户更容易持续积累。数据量越大，系统越智能，形成正向飞轮。
02
形成数据依赖，大幅提升换机成本
用户的生活记录、工作记忆、个人资产都沉淀在小米生态中。数年的记忆积累，是最强的留存壁垒——用户换机的年限将因此显著拉长。
03
记录数据反哺整个系统，做更好的服务
用户行为数据帮助小米提供更精准的智能推荐、生活提醒、行为分析，让整个手机生态的服务质量持续提升。
07 · 挑战

我们面临的三个核心挑战

01 用户行为迁移：体验设计是最难的关

不可能一步到位替代所有用户的使用习惯。从传统笔记到智能笔记，是一次用户心智的迁移，需要循序渐进。

使用上 体验要足够顺滑，不能让用户感到割裂
感知上 每个新版本要让用户感受到明显的智能跃升
传播上 需要有清晰的亮点和卖点，能在社媒上形成自然传播
02 数据层挑战：历史数据 + 跨机型流动

老用户的历史数据迁移、不同机型之间的数据传输、以及整个数据架构的升级改造，都是实际的工程挑战。数据连续性直接决定用户能否真正形成"记忆依赖"。

03 系统能力挑战：多端录入 → MCP 输出

要做到真正的系统级能力，需要打通整个链路：

手机 / 手表 / 车机
多端录入 → Memory 记忆层
统一存储与理解 → MCP 对外输出
系统层能力开放

每一环的稳定性和延迟都会影响整体体验，这是产品成为真正基础设施的必经之路。

08 · 下一步

现在要做的三件事

01 版本规划：v1 聚焦四个核心目标

从两个维度同时拆分版本路径：用户体验的感知跃升 + 研发能力的实际排期。v1 不求大而全，聚焦让用户第一次打开就能感受到"这个笔记不一样"。

1 多端无摩擦输入，语音优先
把语音提到第一位，大幅降低用户输入门槛。
🎙️ 语音输入（核心入口） 📞 通话录音自动进入笔记 🤝 会议录音自动整理 📱 屏幕读屏 / 上下文感知记录
2 笔记对话：让用户感受到 AI 记忆
用户第一次打开就能感知到——这个笔记是 AI 的，因为可以直接和它对话、修改内容。
💬 与笔记内容直接对话 ✏️ 通过对话修改和补充记录
3 内容生成
基于已有记录，AI 自动生成结构化内容，让记录产生直接价值。
📝 会议纪要自动生成 📖 学习总结 / 行动计划 🔄 项目复盘自动整理
4 Ask AI：自然语言查询记忆库
用自然语言直接问，系统从个人记忆库中找答案，用完即抛。
❓ 我今天开了什么会？ ❓ 我记过那家餐厅吗？ ❓ 上次和这个客户谈了什么？
02 产品设计启动：系统打通 + 交互定义 + 跨团队拉通

核心交互流程定义：记录、查询、生成三条主线
与小米其他系统（小爱、系统搜索等）的协作团队拉通
03 技术方案确认：迁移路径 + 数据架构
现有技术与数据的迁移方案制定
Memory 层的 database 基础架构设计
与技术团队对齐最终方案，确定技术边界与依赖项
