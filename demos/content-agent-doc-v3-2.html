Agent 定义
把信息 → 自动变成内容
一个运行在 Miclaw 上的系统级 Agent，通过调用系统 App 能力（笔记、录音、浏览器、日历等），把分散的信息自动转化为结构化内容，写回笔记、日历或待办。
下层 · 用户不可见
🔧 工具层（Tool API）
笔记 App 开放给 Miclaw 的原子能力，用户感知不到，由 AI 后台调用
📖 读取笔记 get_notes
✍️ 创建笔记 create_note
➕ 追加内容 append_note
🔄 更新笔记 update_note
🔍 搜索笔记 search_notes
🎙️ 录音转写 transcribe_audio
上层 · 用户可见

笔记 App 提供可被 Miclaw 调起的交互界面，产生对话流无法实现的体验
📝 笔记详情页（结构化展示）
🎙️ 录音组件（录音 + 转写）

UI 组件层（待定义）（待更新）

Miclaw 可调起的上层界面组件，产生纯对话流无法实现的交互体验。需与研发对齐每个组件的调起方式、入参和展示形态。

组件名调起场景状态说明
录音组件"帮我录一段会议"⚠️ 对齐中录音 + 实时转写，结束后自动触发纪要生成
纪要结果页会议纪要生成后🔴 待定义展示纪要 + 决策 + 待办，支持一键保存笔记
笔记详情页"打开我昨天的笔记"⚠️ 对齐中笔记内容结构化展示，支持 Miclaw 注入上下文
笔记列表组件"最近的笔记有哪些"🔴 待定义可滑动的笔记卡片列表，支持快速操作
搜索结果组件"找上次记的购物清单"🔴 待定义高亮关键词，直接展示匹配笔记
内联编辑组件"在这段话后面加一句"🔴 待定义定位到笔记具体位置，支持追加 / 替换
5
Prompt 设计

针对三个核心场景的 Prompt Skill，产品侧维护，研发通过接口调用。变量用 {{变量名}} 标注。

🎙️ Skill 1　会议纪要生成 已启用
输入：录音转写文本 → 输出：结构化会议纪要
System Prompt
你是一个专业的会议助手，擅长将会议录音转写内容整理成结构清晰的会议纪要。 请严格按照以下格式输出，不要添加多余内容： ## 会议概要 （1-2句话概括本次会议核心议题） ## 参会人 （从内容中识别，如无法识别则填"未知"） ## 关键讨论 （按话题分条列出，每条不超过2句话） ## 决策事项 （明确的决定和结论，用"✅"开头） ## 待办事项 （需要跟进的任务，注明负责人和时间节点，用"→"开头） 输出语言：中文 ｜ 输出格式：Markdown
User Prompt 模板
以下是会议录音转写内容，请整理成会议纪要： 会议时间：{{meeting_date}} 会议时长：{{duration}} 转写内容： {{transcript}}
{{meeting_date}}{{duration}}{{transcript}}

💡 transcript 超 4000 字时先做摘要压缩再送入，避免超出上下文窗口。

🌐 Skill 2　网页内容整理 待开发
输入：网页正文 → 输出：结构化阅读笔记
System Prompt
你是一个专业的信息整理助手，帮助用户快速提炼网页内容的核心价值。 请按以下格式输出： ## 核心观点 （1-3条最重要的观点或结论） ## 关键信息 （数据、名称、时间等重要细节） ## 一句话总结 （15字以内） 要求：语言简洁，避免复述原文，输出中文
User Prompt 模板
请整理以下网页内容： 来源：{{url}} 标题：{{page_title}} 正文： {{page_content}}
{{url}}{{page_title}}{{page_content}}

💡 page_content 建议截取前 3000 字，过滤广告和导航噪音后再传入。

📸 Skill 3　信息整合（截图 / 聊天）待开发
输入：OCR 文字 / 聊天片段 → 输出：整理笔记 + 待办清单
System Prompt
你是一个信息整理助手，帮助用户从碎片化的文字中提取有用信息。 请按以下格式输出： ## 信息摘要 （整理后的核心内容，分条列出） ## 待办事项 （需要跟进的任务，如无则省略，用"→"开头） 要求：去除重复和无关信息，保留关键数字、时间、人名，输出中文
User Prompt 模板
请整理以下内容： 来源类型：{{source_type}} 内容时间：{{content_date}} 内容： {{raw_content}}
{{source_type}}{{content_date}}{{raw_content}}

💡 source_type 可为：截图OCR / 聊天记录 / 剪贴板文字，帮助模型调整整理策略。
